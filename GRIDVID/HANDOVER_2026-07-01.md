## 1. THE OBJECTIVE (locked, in memory)
The DSL is the LANGUAGE a **consumer, NON-agentic LLM (Qwen-30B-A3B)** GENERATES PROGRAMS in (INDUCTION). Corpus =
(task → DSL-program) labelled pairs; `meta.program` is the LABEL. At test the model writes a program → execute →
verify. **Everything must stay generable by a non-agentic model writing TEXT-DSL; the engine verifies.**

## 2. The real system (4 DSL layers — do not reinvent)
- **engine.js** — the scene-DSL parser+verifier (`buildTask`, `validateTask`, `runScene`, `SHAPES`, grammar via
  `node cli.js dsl`). What Qwen writes. Has DORMANT grid-ops to expose: `solveLatin`, `markEnclosedRegions`,
  `isConnected`, `zoomGrid`, `rotate/flip/transformCells`.
- **program.js** — the typed object-combinator layer (`applyNode` executor; selectors × keys × regions ×
  per-object transforms incl. relational anchors × combinators seq/parallel/mask/dispatch/bind/repeat). The
  induction LABEL `meta.program.tree`. **This is where the deep-rule work lives.**
- **gridgen.js** — ~30 verified families + the `task <family>` menu spec-DSL (robust; the LLM only picks).
- **DSL/** (Python) — `arc_dsl/core.py`, `arcworld.py`, `arcvm.py` — a research spike, disconnected. `DSL/task.py`
  loads the 120 REAL eval tasks.

## 3. KEY STRATEGIC CONCLUSIONS (hard-won this session — do not relitigate)
1. **Weak-LLM FREE-AUTHORING is a DEAD END for the corpus.** A one-shot Haiku × 20 test produced nonsense that
   passed the old gates (empty outputs, degenerate tests, trivial changes). The corpus = **program-first**
   (correct-by-construction: `program.js` legend/dispatch/anchor/containment + trace/NL/DSL). The LLM is **taste /
   proposer** only (pick from the menu, or propose compositions/shapes the engine instantiates). Matches the KB
   "reconciliation" nodes.
2. **Complexity = the DEPTH of ONE rule, NOT the count of parallel rules.** Depth = a **derived intermediate**
   computed from the grid and CONSUMED later in the SAME rule (legend map / containment relation / bands / count).
   The metric is `Dchain` (longest data-dependency chain), rule-count terms deleted.
3. **Step-by-step = the execution TRACE (thinking-grids), NOT GIFs** for the solve tier. The `seq` fold already
   computes every intermediate scene; `runWithTrace` captures them. GIFs stay only for the physics/dynamics tier.
4. **Selection is a REAL grid colour** (highlight value **10 = white**) painted into the thinking-grid at the
   derive/selection step — a human "pointing" at what it reasons about. NOT a CSS overlay.
